# Schema

This file defines the wiki's page conventions, frontmatter rules, and naming standards.
Edit it to match your domain. The defaults below follow the
[Karpathy LLM Wiki](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f) pattern.

## Page Types

| Type | Description | Examples |
|:---|:---|:---|
| **Concept** | A single idea, pattern, or principle | `distributed-consensus.md`, `eventual-consistency.md` |
| **Entity** | A system, tool, or technology | `apache-kafka.md`, `postgresql.md` |
| **Synthesis** | Cross-page analysis (auto-generated by /query) | `consensus-tradeoffs.md` |
| **Context** | Summary of an ingested context document | `ddia-chapter-5.md` |

## Naming Convention

- **kebab-case** for all filenames: `distributed-consensus.md`
- **Single topic per page** ŌĆö if a section grows too large, split into its own page
- **Subdirectories optional** ŌĆö use when wiki exceeds ~50 pages: `wiki/pages/concepts/`, `wiki/pages/entities/`

## Required Frontmatter

Every wiki page must include:

```yaml
---
title: Page Title
description: One-line summary of the page
tags: []
contexts: []
created: YYYY-MM-DD
updated: YYYY-MM-DD
---
```

### Frontmatter Fields

| Field | Required | Description |
|:---|:---:|:---|
| `title` | Yes | Page title (must match `# Heading`) |
| `description` | Yes | One-line summary (used in index.md and search) |
| `tags` | Yes | Category tags from taxonomy below |
| `contexts` | Yes | Paths to context files (relative to workspace root) |
| `aliases` | No | Alternate names, abbreviations, translations (improves wikilink matching) |
| `status` | conditional | Required for issue/bug pages: `open` / `resolved` / `wontfix` |
| `created` | Yes | Creation date |
| `updated` | Yes | Last update date |

## Wikilinks

Use `[[page-name]]` for every entity that has (or should have) its own page.
Linked pages that don't exist yet become "wanted pages" ŌĆö a useful signal for what to write next.

GOOD: "Apache Doris uses [[raft-consensus]] for replication"
BAD:  "Apache Doris uses Raft for replication" (no wikilink, loses the connection)

## Related Section

Every page should end with a `## Related` section:

```markdown
## Related

- [[other-page]] ŌĆö one-line relationship description
- [[another-page]] ŌĆö how it relates
```

## Tags

Define your tag taxonomy here as the wiki grows. Start minimal, expand as needed.

Example:
- `concept` ŌĆö abstract ideas and patterns
- `system` ŌĆö concrete tools and technologies
- `method` ŌĆö processes and workflows
- `synthesis` ŌĆö cross-page analysis (auto-generated)
